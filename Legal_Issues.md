# Legal Issues

TeamMatch may involve legal concerns related to student data, privacy, intellectual property, and third-party software usage. Since the application handles student information, the system should follow privacy expectations and only collect data needed for team formation.

The application should avoid storing unnecessary personal information. Student records such as names, emails, skills, availability, and team assignments should be protected and used only for the course. If used in an educational setting, the team should be aware of student privacy rules and institutional policies.

TeamMatch may use third-party technologies such as FastAPI, Python, JavaScript, Azure, AWS, or other libraries. The team should verify that these tools are open-source or properly licensed for educational and project use. Any external APIs, frameworks, or datasets used in the project should be cited and reviewed for licensing restrictions.

The application should not allow users to upload copyrighted content, steal information, or misuse the system. If file uploads are added in the future, the system should restrict file types, scan uploads, and clearly state acceptable use rules.

Overall, the legal plan for TeamMatch is to use properly licensed tools, limit data collection, protect student information, and avoid using copyrighted or unauthorized materials.
